# 即梦视频队列工具操作文档

这份文档给“没有编程经验的人”使用。你只需要照着步骤打开页面、填写内容、点击按钮即可。

## 这个工具是做什么的

即梦生成视频时经常需要排队，而且通常不适合同时生成多个视频。

这个工具的作用是：

1. 你先把多个视频任务加到队列里。
2. 点击“启动队列”。
3. 工具会等第 1 个视频生成完成后，自动开始第 2 个。
4. 后面的任务也会一个接一个自动生成，不需要人工盯着。

## 使用前准备

使用电脑需要提前准备好：

1. 已经安装即梦 CLI。
2. 已经登录即梦 CLI。
3. 电脑上有 Python 3。
4. 已经拿到这个项目文件夹。

如果不确定有没有登录即梦 CLI，可以先让懂技术的人帮你确认一次。

## 第一次打开工具

### 第 1 步：打开终端

Mac 上操作：

1. 按 `Command + 空格`。
2. 输入“终端”。
3. 回车打开。

### 第 2 步：进入项目文件夹

在终端里输入 `cd `，注意 `cd` 后面有一个空格。

然后把这个项目文件夹拖到终端窗口里，终端会自动填入路径。

最后按回车。

示例看起来类似这样：

```bash
cd /Users/你的用户名/Downloads/jimeng-cli
```

### 第 3 步：启动页面

在终端里输入：

```bash
python3 web_app.py
```

看到类似下面这行，就说明启动成功：

```txt
Dreamina Queue UI running at http://127.0.0.1:8765
```

### 第 4 步：打开浏览器页面

打开浏览器，访问：

```txt
http://127.0.0.1:8765
```

文生视频页面是：

```txt
http://127.0.0.1:8765/text2video
```

注意：工具运行期间，不要关闭终端窗口。

## 页面入口说明

页面顶部有两个入口：

- `全能参考`：适合上传图片、视频、音频作为参考来生成视频。
- `文生视频`：适合只输入文字提示词，不上传素材，直接生成视频。

两个页面共用同一个队列。无论在哪个页面添加任务，都会排到同一个队列里按顺序执行。

## 文生视频怎么用

适合场景：你只有一段文字描述，想直接生成视频。

操作步骤：

1. 打开 `http://127.0.0.1:8765/text2video`。
2. 在“视频段名称”里写一个好识别的名字，比如 `片段1-开场`。
3. 在“提示词”里写你想生成的视频内容。
4. 设置“时长”，比如 `5` 或 `6`。
5. 设置“比例”，常用竖屏是 `9:16`。
6. 在“模型”里选择模型。
7. 点击“加入队列”。

加入后，下面“队列内容”里会出现这条任务。

## 全能参考怎么用

适合场景：你有图片、视频或音频，希望即梦参考这些素材生成视频。

操作步骤：

1. 打开 `http://127.0.0.1:8765`。
2. 在“视频段名称”里写名字。
3. 在“提示词”里写视频描述。
4. 选择时长、比例、模型。
5. 上传图片、视频或音频。
6. 点击“加入队列”。

工具会根据你上传的素材自动选择生成方式：

- 只有 1 张图片：按图生视频处理。
- 多张图片：按多图分镜处理。
- 图片、视频、音频混合：按全能参考处理。

## 上传图片和粘贴图片

上传图片：

1. 点击“选择文件”。
2. 选择本地图片。
3. 点击“上传图片并填入”。

粘贴图片：

1. 先复制一张图片或截图。
2. 点击页面里的“把焦点点到这里，然后直接粘贴截图或复制的图片”区域。
3. Mac 按 `Command + V`，Windows 按 `Ctrl + V`。

上传后的素材会显示在“已上传素材”区域。

## 提示词里怎么用 @ 素材

上传素材后，页面会自动生成类似这样的引用：

```txt
@Image角色正面
@Video参考动作
@Audio背景音乐
```

使用方式：

1. 在提示词中输入 `@`。
2. 页面会弹出素材列表。
3. 点击你要引用的素材。
4. 工具会把引用插入到提示词里，并自动把素材路径加入对应素材输入框。

注意：这里的 `@` 是这个工具里的本地辅助写法，不是即梦 CLI 官方语法。真正提交前，工具会自动转换成普通文字和素材路径。

## 队列内容怎么管理

加入队列后，每一条任务都会显示成一张卡片。

你可以做这些操作：

- 修改“视频段名称”。
- 修改“提示词”。
- 展开“素材与参数”，修改图片、视频、音频、时长、比例、模型。
- 点击“复制”，复制一条类似任务。
- 点击“删除”，删除这一条任务。

删除或修改后会自动保存到本地文件，不需要再手动点“保存队列”。

队列文件位置是：

```txt
.webui/web.queue.json
```

如果你想确认是否真的删除了，可以打开这个文件查看，里面的 `segments` 数组就是当前队列。

## 启动队列

添加好任务后：

1. 确认队列内容没问题。
2. 点击“自动检测 dreamina”。
3. 如果检测成功，页面会填好 dreamina 路径。
4. 点击“启动队列”。

启动后，工具会自动按顺序生成视频。

## 停止队列

如果想中途停止：

1. 点击“停止队列”。
2. 等运行状态变成空闲。

停止后，已经完成的任务不会丢。

## 断点续跑

如果中途关闭了电脑、断网、或者手动停止了队列，下次可以勾选：

```txt
断点续跑
```

然后再点“启动队列”。

这样已经成功的任务会跳过，继续跑后面的任务。

## 输出文件在哪里

默认输出目录是：

```txt
web-output
```

常见文件：

- `web-output/queue-state.json`：记录每个任务是否成功、失败原因、submit_id。
- `web-output/logs/`：每个任务的运行日志。
- `web-output/任务名称/`：生成成功后下载的视频结果。

## 常见问题

### 1. 页面打不开

先确认终端里 `python3 web_app.py` 还在运行。

如果终端已经关闭，需要重新执行：

```bash
python3 web_app.py
```

### 2. 提示端口被占用

可以换一个端口启动：

```bash
python3 web_app.py --port 8766
```

然后浏览器打开：

```txt
http://127.0.0.1:8766
```

### 3. 提示 dreamina 命令不存在

说明电脑找不到即梦 CLI。

可以先点页面里的“自动检测 dreamina”。

如果还是失败，需要让懂技术的人确认即梦 CLI 是否安装，并把正确路径填到“dreamina 路径”里。

### 4. 删除任务后又出现了

请刷新页面确认。

当前版本删除会自动保存到：

```txt
.webui/web.queue.json
```

如果你打开的是旧文件：

```txt
.webui/web.queue.txt
```

那不是当前默认队列文件，不要再看这个旧文件。

### 5. 生成失败了怎么办

先看“运行面板”里的失败原因。

常见处理方式：

- 提示词太复杂：删短一点。
- 素材太多：减少图片或视频数量。
- 时长太长：改成 4 到 6 秒。
- 账号需要确认授权：去即梦网页端处理授权后再继续。

### 6. 可以关闭浏览器吗

可以关闭浏览器，但不要关闭终端窗口。

真正跑队列的是终端里的程序，浏览器只是操作页面。

## 给别人使用时怎么交付

把整个项目文件夹发给对方即可。

建议不要一起发这些内容，除非你确定里面没有隐私：

- `.webui/uploads`
- `web-output`
- `.webui/web.queue.json`

如果你想给对方一个干净版本，可以只保留代码文件，让对方自己上传素材、自己登录即梦 CLI。

## 最简使用流程

如果只记一套流程，就记这个：

1. 打开终端。
2. 进入项目文件夹。
3. 运行 `python3 web_app.py`。
4. 浏览器打开 `http://127.0.0.1:8765`。
5. 选择“文生视频”或“全能参考”。
6. 填内容，点“加入队列”。
7. 确认队列没问题，点“启动队列”。
8. 去 `web-output` 文件夹里找生成结果。
