# 视频目录结构说明

## 📁 新的目录结构

从现在开始，下载的视频将按以下结构组织：

```
douyin_videos/
├── 账号名1/
│   ├── 视频标题1/
│   │   ├── [1w赞]_视频标题1_7542097231621950772.m4a
│   │   ├── [1w赞]_视频标题1_7542097231621950772.json
│   │   └── [1w赞]_视频标题1_7542097231621950772.transcript.json
│   ├── 视频标题2/
│   │   ├── [5000赞]_视频标题2_7542097231621950773.m4a
│   │   ├── [5000赞]_视频标题2_7542097231621950773.json
│   │   └── [5000赞]_视频标题2_7542097231621950773.transcript.json
│   └── ...
├── 账号名2/
│   ├── 视频标题3/
│   │   └── ...
│   └── ...
└── 未知账号/  ← 如果无法获取账号名
    └── ...
```

## 🎯 优势

### 1. 按账号分类
- 清楚知道每个视频来自哪个账号
- 方便分析某个账号的所有视频
- 便于管理和清理

### 2. 按视频标题分组
- 每个视频独立目录
- 所有相关文件（音频、元数据、转录）在一起
- 避免文件名过长问题

### 3. 便于审核
- 审核界面会显示账号名和目录信息
- 更容易识别视频来源

## 🔄 兼容性

### 旧数据
- 旧的目录结构（按标签分类）仍然可以正常审核
- 审核工具会自动识别两种结构

### 新数据
- 从现在开始下载的视频使用新结构
- 审核界面会显示账号名和目录信息

## 🛠️ 技术细节

### 文件名清理规则
- **账号名**：最多 30 个字符
- **视频标题**：最多 50 个字符
- 自动移除非法字符：`/ \ : * ? " < > |`

### 特殊情况处理
- 如果无法获取账号名 → 使用 `未知账号`
- 如果无法获取标题 → 使用 `视频_{video_id}`

## 🚀 使用方法

### 下载新视频
```bash
# 使用现有的下载脚本，会自动使用新结构
python douyin/douyin_cli.py
```

### 审核视频
```bash
# 审核工具自动兼容新旧结构
python start_review.py
```

### 导入知识库
```bash
# 导入工具自动兼容新旧结构
python douyin/import_videos.py douyin_videos
```

## 💡 建议

1. **新下载的视频**会自动使用新结构
2. **旧视频**可以保持原样，不需要迁移
3. **审核工具**兼容两种结构
4. 如果想统一结构，可以手动整理旧视频
