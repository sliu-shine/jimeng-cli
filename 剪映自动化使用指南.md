# 剪映导入包自动化使用指南

## 当前方案

新版剪映专业版的本地草稿不是普通 JSON 工程。真实草稿目录里包含私有编码文件，例如 `draft_info.json`、`template-2.tmp`、`Timelines/`、`Resources/` 等，直接手写简化版 `draft_content.json` 无法稳定出现在剪映首页。

因此本项目已切换为更稳定的方案：

```text
自动生成视频片段
  ↓
合并为 final.mp4
  ↓
生成 subtitles.srt
  ↓
导入剪映继续精修
```

输出目录示例：

```text
jianying_import_packages/自动生产视频_1778060000/
├── final.mp4          # 合并后视频
├── subtitles.srt      # 可导入剪映的字幕
└── import_guide.json  # 片段、时间轴、源视频记录
```

## Web 方式

启动服务：

```bash
python3 jianying_web_server.py
```

浏览器打开：

```text
http://127.0.0.1:7862
```

操作流程：

1. 输入项目名称。
2. 输入视频文案。
3. 勾选要合并的视频片段。
4. 保持「生成 SRT 字幕文件」开启。
5. 点击「生成剪映导入包」。

生成完成后，会显示导入包路径、`final.mp4` 和 `subtitles.srt` 路径。

## 命令行方式

```bash
python3 jianying/auto_workflow.py \
  --transcript "你的视频文案" \
  --videos "./web-output/**/*.mp4" \
  --project-name "自动生产视频"
```

从文案文件读取：

```bash
python3 jianying/auto_workflow.py \
  --transcript ./script.txt \
  --videos "./videos/*.mp4"
```

不生成字幕：

```bash
python3 jianying/auto_workflow.py \
  --transcript ./script.txt \
  --videos "./videos/*.mp4" \
  --no-subtitles
```

## 导入剪映

1. 打开剪映专业版。
2. 点击「开始创作」。
3. 导入生成的 `final.mp4`。
4. 导入 `subtitles.srt`，或使用剪映自动识别字幕。
5. 在剪映里继续添加配音、滤镜、转场、封面和导出。

## 注意事项

- 需要安装 `ffmpeg`。
- 如果多个视频编码一致，工具会优先无损合并。
- 如果无损合并失败，工具会自动转码为 H.264/AAC。
- 这个方案生成的是稳定媒体包，不是剪映多轨草稿；它更适合批量生产后进剪映做最后精修。

安装 ffmpeg：

```bash
brew install ffmpeg
```
